# Track Collector+ для Яндекс Музыки

Скрипт для экспорта треков из плейлистов веб-версии Яндекс Музыки через консоль браузера.

Основной сценарий — выгрузить плейлист «Мне нравится» в формат TXT или CSV, чтобы дальше работать с ним где угодно (резервная копия, анализ, перенос в другой сервис и т.п.).

> ⚠️ Скрипт не использует никаких приватных API. Он просто читает уже отрендеренный HTML на странице Яндекс Музыки в вашем браузере.

---

## Возможности

- Экспорт треков из любого плейлиста веб-версии Яндекс Музыки (в том числе «Мне нравится»).
- Поддержка больших плейлистов (несколько сотен треков).
- Сбор:
  - названия трека;
  - артистов;
  - идентификатора трека (если доступен).
- Работа с «нестандартными» строками плейлиста:
  - когда у трека нет обычной ссылки /track/...;
  - когда данные спрятаны в aria-label и alt атрибутах.
- Выгрузка в файлы:
  - tracks-full.txt — формат Артист – Название;
  - tracks-full.csv — колонки Artist, Title, Source.

## Как использовать

1. Откройте плейлист в Яндекс Музыке

   Зайдите в веб-версию Яндекс Музыки в браузере и откройте нужный плейлист, например «Мне нравится».

2. Откройте DevTools → Console

   - Chrome: F12 или Ctrl+Shift+I (Cmd+Option+I на Mac), вкладка Console.

3. Разрешите вставку кода

   В консоли выполните команду:

   ```js
   allow pasting
   ```

   Это стандартное предупреждение Chrome, защищающее от случайного вставления кода. Так мы подтверждаем, что вставляем скрипт осознанно.

4. Вставьте скрипт

   - Откройте файл [ym-track-collector-plus.js](./ym-track-collector-plus.js) в этом репозитории.
   - Скопируйте его содержимое целиком.
   - Вставьте в консоль браузера и нажмите Enter.

5. Прокрутите плейлист

   Справа появится небольшая панель Track Collector+.  
   Теперь медленно прокрутите плейлист вниз до самого конца, чтобы Яндекс подгрузил все треки.

   По мере прокрутки в панели будет расти счётчик треков.

6. Завершите сбор

   Когда долистали до конца (и можно чуть «пошевелить» вверх-вниз):

   - Нажмите кнопку Finalize capture в панели.
   - Подождите, пока статус сменится на Ready to download.

7. Скачайте файлы

   После завершения:

   - Нажмите Download .txt для tracks-full.txt (удобно для чтения и быстрого копирования).
   - Нажмите Download .csv для tracks-full.csv (удобно для импорта в таблицы/скрипты).

## Что делает скрипт внутри

1. Встраивает на страницу панель #ym-scraper со счётчиком и кнопками.
2. Находит контейнер скролла плейлиста (виртуализованный список).
3. На каждый скролл запускает функцию extractTracks() с задержкой (debounce).
4. extractTracks() проходит по строкам треков (.CommonTrack_root__i6shE) и для каждой строки:
   - пытается взять:
     - ссылку /album/.../track/... → trackId;
     - название span.Meta_title__GGBnH;
     - версию/ремикс span.Meta_version__c2sHU;
     - артистов span.Meta_artistCaption__JESZi внутри div.Meta_artists__VnR52;
   - если данные не найдены, включает fallback-режим:
     - берёт название из img[alt^="Трек ..."];
     - берёт артистов из aria-label, вычитая название трека;
     - формирует синтетический ID (fallback::...), чтобы не было дубликатов.
5. Складывает все треки в Map, чтобы не было повторов.
6. По кнопкам Download .txt / Download .csv формирует содержимое и отдает его браузеру как Blob для скачивания.

## Ограничения и нюансы

- Скрипт зависит от текущей DOM-структуры веб-версии Яндекс Музыки:
  - классы вида .CommonTrack_root__..., .Meta_title__..., .Meta_artistCaption__...;
  - атрибуты aria-label, alt и т.п.
- Если Яндекс существенно поменяет верстку, селекторы нужно будет обновить.
- В тестах на большом плейлисте:
  - автоматически собирается ~98% треков;
  - несколько треков могут выпадать, если у них совсем нет привычной структуры (особые карточки, эпизоды и т.п.).
- Скрипт ничего не отправляет наружу:
  - работает только внутри вашего браузера;
  - не использует сетевые запросы к сторонним сервисам.
